# 新机器部署指南（从零开始 · Docker 全栈 + 宿主 Kit）

把整套 **Sim2Real（运营模拟 + Creator + 3D 数字孪生）** 部署到**一台全新的 Windows 机器**上。
形态：**5 个服务全部跑在 Docker 里**（两个后端 + 两个前端 + PostgreSQL），**Omniverse Kit 单独在宿主机原生运行**（要 GPU，不进 Docker）。

> 本指南面向**单机演示**：浏览器、Kit、Docker 都在同一台机。多机/远程访问见末尾「附录 B」。
> **Linux 部署见 [新机器部署指南-Linux.md](新机器部署指南-Linux.md)**（Linux 是这套更顺的目标）。

---

## 0. 总览：这台机器最终会跑起来什么

```
浏览器
 ├─ http://localhost:8080  sim 前端（运营模拟 UI）      ┐
 ├─ http://localhost:8081  aifactory 前端（Creator UI） │  都在 Docker
 ├─ http://localhost:8000  sim 后端（仿真引擎，拥有 DB）│  (compose 5 服务)
 ├─ http://127.0.0.1:8129 aifactory 后端本机调试     │
 ├─ 127.0.0.1:5432         PostgreSQL（共享库，不对 LAN）┘
 └─ (直连) Kit  sim→:8233 ｜ aifactory→:8011  /ov+/kit/playback ┐ 原生跑在宿主机
            WebRTC :12333 / :12334                             ┘ (需 RTX GPU，不在 Docker)
```

| 组件 | 端口（宿主） | 形态 |
|---|---|---|
| sim 前端 | 8080 | Docker（nginx） |
| aifactory 前端 | 8081 | Docker（nginx） |
| sim 后端 | 8000 | Docker（FastAPI，**建表+灌 P9 种子**） |
| aifactory 后端 | 127.0.0.1:8129 | Docker（FastAPI，纯消费共享库） |
| PostgreSQL | 127.0.0.1:5432 | Docker（仅本机可达） |
| Omniverse Kit | sim 用 8233 ｜ aifactory 用 8011 ｜ WebRTC 12333 / 12334 | **宿主原生**（GPU） |

> sim 与 aifactory 前端连的是**两个分开的 Kit /ov 端口**：sim → `:8233`，aifactory → `:8011`。勿混用。

## 1. 前置依赖（这台机器要装什么）

| 必装 | 说明 |
|---|---|
| **Docker Desktop**（WSL2 后端） | 跑那 5 个服务。已验证 Docker 29.x + Compose v5 |
| **NVIDIA RTX GPU + 较新驱动** | Kit 渲染/串流。无 GPU 则 3D 不可用，其余功能不受影响 |
| **Git** | 拉代码 |
| **VS C++ Build Tools（含 C++ 工作负载）** | **仅构建 Kit 时需要** |

> **不用单独装 Node / Python / 后端依赖**——前端在 Docker 里构建、后端在 Docker 里运行，全部由镜像搞定。国内网络也 OK：镜像构建走清华源 / npmmirror。

## 2. 拿代码 + 资产

### 2.1 主仓库（本项目）
把本仓库整个拷/clone 到机器上，例如 `C:\Workspace\Sim2Real-Creator&Simulation`。
> 源码在 `源码/` 目录下（注意是中文名）。两处历史源码 bug（sim 后端 Dockerfile 缺 libc6-dev、aifactory 后端 `result/ResultUtils.py`）已在源码内修好，全新 clone 可直接构建。

### 2.2 Kit App（kit-app-template，单独 clone）
```powershell
git clone -b feat/aifactory-migration https://github.com/chevyhou0117-lgtm/kit-app-template.git C:\Workspace\Sim2Real\kit-app-template
```

### 2.3 资产库（~31GB，不在 git 里，单独拷）
把整个资产目录拷到本机某处，例如 `D:\Sim2Real\storage`。它**顶层应包含**：
```
storage/
├── thumbnails/      # 缩略图（aifactory 前端用）
├── Library/         # 设备 USD
├── Line_Library/    # 线体 USD 
└── Data/ , MBD_NV/  # 全厂场景 USD 等
```
**记住这个路径**（下面 Kit 和 Docker 都要指向它），本指南用 `<STORAGE>` 代指（如 `D:/Sim2Real/storage`）。

## 3. 构建并启动 Kit（GPU，Docker 之外）

```powershell
cd C:\Workspace\Sim2Real\kit-app-template
.\repo.bat build           # 首次会用 packman 联网拉 Kit SDK 109.0.3（数 GB），需外网/内网缓存 + VS Build Tools
```

### 3.1 让 Kit 指向资产库
设置环境变量 `AIFACTORY_USD_ROOT=<STORAGE>`（Kit 从这里读 USD）。
（在 `source/extensions/aifactory.service.setup/.../.env` 里，或系统环境变量。）

Kit 的 aifactory 扩展只读完整的 `AIFACTORY_DATABASE_URL`，宿主原生模式应设为：

```text
AIFACTORY_DATABASE_URL=postgresql+asyncpg://postgres:<URL_ENCODED_PASSWORD>@127.0.0.1:5432/aifactory_simulation
```

其中密码必须与 `docker/.env` 的 `POSTGRES_PASSWORD` 一致。

### 3.2 启动 Kit
```powershell
.\repo.bat launch
# 选 fii.houyiming_streaming.kit
```
起来后应监听 Kit 的 /ov + /kit/playback 控制端口 —— **sim 前端用 `:8233`、aifactory 前端用 `:8011`（两者分开）** —— 以及 `:12333`（media）、`:12334`（signal）。

## 4. 配置并启动 Docker（5 个服务）

### 4.1 配置 `docker/.env`
```powershell
cd C:\Workspace\Sim2Real-Creator&Simulation
Copy-Item docker\.env.demo.example docker\.env
```
打开 `docker\.env`，按本机改这几项（单机演示下大多保持默认即可）：
```bash
POSTGRES_PASSWORD=postgres                  # 演示可默认；正式部署改强密码
AIFACTORY_STORAGE_HOST=D:/Sim2Real/storage  # ★ 指向 §2.3 的 <STORAGE>（缩略图/USD 从这挂 /static）
CREATOR_AUTH_USERNAME=creator               # Creator :8081 的 Basic Auth 用户名
CREATOR_API_KEY=<64位随机十六进制值>         # 必填；可用 `openssl rand -hex 32` 生成，同时保护后端写接口

# ★ 跨机/远程访问唯一必改项：Kit 宿主的【局域网可达 IP】（单机演示留 127.0.0.1）。
#   运行期注入——容器启动时据此生成 runtime-config.js，浏览器据此连 WebRTC / 调 Kit。
#   改它后只需 up -d（无需 rebuild）。详见附录 B。
KIT_HOST_IP=127.0.0.1
KIT_API_PORT=8233                           # sim 前端的 Kit /ov 控制端口
BACKEND_PORT=8000                           # sim 后端发布端口（Kit 回拉事件用）
KIT_API_UPSTREAM=host.docker.internal:8011 # Windows 宿主原生 Kit；Linux 容器 Kit 则用 sim2real:8011
AIFACTORY_CORS_ORIGINS=http://localhost:8081,http://127.0.0.1:8081

# 以下 VITE_* 仅作【构建期默认/回退】，会被上面 KIT_HOST_IP 的运行期值覆盖，单机演示保持默认即可
VITE_KIT_API_URL=http://localhost:8233      # sim 前端的 Kit /ov + /kit/playback（回退默认；与 aifactory 的 8011 分开）
VITE_KIT_STREAM_URL=enabled                 # 非空=启用「直连 WebRTC」（已不需要 5183 串流页）
VITE_BACKEND_DIRECT_URL=http://localhost:8000
VITE_KIT_STREAM_MODE=direct                 # direct=直连 WebRTC；iframe=回退 5183（逃生门）
```

访问 Creator `:8081` 时浏览器会弹出登录框，密码就是 `CREATOR_API_KEY`。真实密钥只放在未跟踪的
`docker/.env`，不要写入前端源码或提交到 Git。
Basic Auth 本身不加密凭据；跨机器访问 `:8081` 时必须由前置反向代理提供 HTTPS，或限制在可信内网/VPN。

### 4.2 对齐全厂场景 USD 的绝对路径（首次灌库前）
`md_creator_project.creator_url` 是**绝对路径**，由 `源码/sim_backend/seed_data/creator_project.csv` 灌入，默认值是别的机器的路径。改成本机全厂 USD 的真实绝对路径，例如：
```
creator_url 改为：  D:/Sim2Real/storage/Data/P9_animations/Houston_F_NV/demo520.usd
```
> 若已经灌过库再改，需 `docker compose ... down -v` 清库重灌，或直接改数据库里的 `creator_url`。

### 4.3 构建 + 启动
```powershell
docker compose -f docker/docker-compose.demo.yml build      # 首次构建较慢（拉依赖 + 前端 npm ci）
docker compose -f docker/docker-compose.demo.yml up -d
docker compose -f docker/docker-compose.demo.yml logs -f sim-backend   # 看迁移 + 灌 P9 种子日志
```
启动链：`sim-postgres` 健康 → `sim-backend`（**alembic 迁移 + 首次灌 P9 种子**，含 aifactory 的 creator_tables）→ `aifactory-backend`（纯消费）+ 两个前端。

> **首次** `up` 会建表 + 灌 P9 种子；之后 `up`/`restart`/`down→up` **都不会重灌**（按 md_factory 是否为空门控 + 数据在持久卷 `sim_pgdata`）。只有 `down -v`（删卷）才会重新迁移+灌。

## 5. 启动顺序与验证

**启动顺序**：先 `docker compose up -d`（PG→后端→前端会按依赖自动排序）→ 再确保 **Kit 已在宿主跑起来**。

| 检查 | 方法 | 预期 |
|---|---|---|
| sim 后端 | `http://localhost:8000/docs` | Swagger |
| aifactory 后端 | `http://127.0.0.1:8129/docs` | Swagger（LAN 直连应失败） |
| 全部容器 | `docker compose -f docker/docker-compose.demo.yml ps` | 5 个都 healthy |
| sim 前端 | `http://localhost:8080` | 页面渲染、接口不报错 |
| aifactory 前端 | `http://localhost:8081` | 资产库列表 + 缩略图能显示 |
| 3D 串流 | sim 前端进某方案 → 跑模拟 → 3D 回放页 | 视口出画面、随窗口自适应 |

## 6. 常用操作

```powershell
# 停（保留数据）
docker compose -f docker/docker-compose.demo.yml down
# 停并清库（下次 up 重新迁移 + 灌 P9）
docker compose -f docker/docker-compose.demo.yml down -v
# 改了前端代码 / stream.config.json 默认值 / 任何 VITE_*（构建期烤入）→ 必须重 build 再起
docker compose -f docker/docker-compose.demo.yml up -d --build sim-frontend
# 只改了 Kit IP（docker/.env 的 KIT_HOST_IP）→ 运行期注入，不用 build，重建容器即可
docker compose -f docker/docker-compose.demo.yml up -d sim-frontend aifactory-frontend
# 只改了 docker/.env 里其它运行时项（如 AIFACTORY_STORAGE_HOST）→ 不用 build，重建容器即可
docker compose -f docker/docker-compose.demo.yml up -d aifactory-backend
```
> `restart` 只重启进程，**不换镜像、不应用 compose 改动**——要应用改动永远用 `up -d`（必要时加 `--build`）。

## 7. 已知限制与排障

- **「重启 Kit」按钮已移除**：容器化后该按钮触发的 Kit 重启会丢失 WebRTC 串流（需手动刷新页面才恢复），效果不佳，已删。Kit 卡死时直接 `docker restart <kit 容器名>` 即可。
- **3D 黑屏**：① 确认 Kit 在跑且 `:8233`、`:12333/:12334` 在监听；② sim 前端（:8080）是
  **跨域**直连 Kit，需 Kit 扩展 CORS 放行 `http://localhost:8080`；③ `KIT_API_UPSTREAM` 为
  `host.docker.internal:8011`；④ `KIT_HOST_IP` 指向浏览器可达的 Kit 宿主。
- **3D 只开场景不动画**：sim 后端 `:8000` 是否已发布到宿主（compose 已发布）、`VITE_BACKEND_DIRECT_URL` 是否宿主可达。
- **aifactory Creator 3D 报 `authenticate/accessToken`**：把 `源码/aifactory_frontend/.../composer/AppStream.tsx` 里 local 配置的 `authenticate` 改 `false`，重 build `aifactory-frontend`（sim 前端已是 false）。
- **缩略图 404**：`AIFACTORY_STORAGE_HOST` 没指对，或资产目录缺 `thumbnails/`。
- **构建报路径错误（中文 `源码/`）**：Docker 一般能解析 UTF-8；若不行，给 `源码` 建 ASCII junction（`mklink /J codes 源码`）并改 compose 的 `context`。

## 附录 A：端口被占用 / 改端口
compose 里 `ports: "宿主:容器"`，改左边宿主端口即可（如 `"18080:80"`）。注意改了后端宿主端口要同步改 `VITE_BACKEND_DIRECT_URL` 并重 build 前端。

## 附录 B：远程 / 多机访问（非单机演示）
浏览器不在 Kit 这台机时 `localhost` 失效。**现在只需一个变量 + 重建容器，无需 rebuild、无需手改 `stream.config.json`**：

1. 编辑 `docker/.env`，把 `KIT_HOST_IP` 设为 **Kit 宿主的局域网可达 IP**（如 `192.168.1.50`）。
2. 重建两个前端容器（env 变了 compose 会自动 recreate → 启动脚本据 `KIT_HOST_IP` 重写 `runtime-config.js`）：
   ```powershell
   docker compose -f docker/docker-compose.demo.yml up -d sim-frontend aifactory-frontend
   ```
3. 验证：浏览器访问 `http://<宿主IP>:8080/runtime-config.js` 应看到该 IP；或 devtools 控制台敲 `window.__RUNTIME_CONFIG__`。
4. 基础设施：确保 `:12333/:12334`、sim 的 `:8233`、`:8000` 从客户端可达。Creator 的
   Kit `:8011` 由 nginx 同源代理，无需向客户端放行；`5432` / `8129` 也不应对 LAN 放行。

> **原理**：Kit IP 是【运行期注入】的——容器启动时 nginx 脚本 `docker-runtime-config.sh` 按 `KIT_HOST_IP` 生成 `/runtime-config.js`，前端优先读 `window.__RUNTIME_CONFIG__`，回退到构建期默认（`stream.config.json` 的 `server` / `VITE_*`，均为 `127.0.0.1`/`localhost`）。故**首次部署 build 一次即可，以后换机器只改 `KIT_HOST_IP` 再 `up -d`**。
> 例外：`VITE_KIT_STREAM_MODE=iframe` 逃生门模式下 `VITE_KIT_STREAM_URL`（5183 页地址）仍是构建期值，需要时才重 build。
